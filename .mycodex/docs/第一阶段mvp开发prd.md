# 第一阶段 MVP 开发 PRD

## 一、产品名称

轻记账。

## 二、产品定位

轻记账是一款面向个人用户的本地记账 App，主要用于记录日常收入和支出，查看本月收支概况，并通过月度预算提醒帮助用户控制消费。

第一阶段不追求复杂图表和导出功能，重点完成“记账 -> 保存 -> 展示 -> 汇总 -> 预算提醒”的核心闭环。

## 三、目标用户

目标用户包括学生、普通上班族和有日常记账需求的个人用户。

典型使用场景：

- 用户午饭花费 18 元，打开 App 记录一笔餐饮支出。
- 用户收到生活费 1000 元，记录一笔收入。
- 用户月底查看本月花了多少钱，还剩多少预算。
- 用户通过账户区分现金、银行卡和信用卡消费。

## 四、MVP 目标

第一阶段目标是完成一个本地可用的个人记账 App。

用户可以：

- 新增收入记录。
- 新增支出记录。
- 选择账单分类。
- 选择支付账户。
- 查看最近账单。
- 查看本月收入、本月支出、本月结余。
- 设置月度预算。
- 查看预算使用进度和预算预警。

## 五、功能范围

### 5.1 首页

首页用于展示本月财务概览和最近账单。

页面内容：

- 本月收入。
- 本月支出。
- 本月结余。
- 月度预算进度。
- 预算状态提示。
- 最近账单列表。

显示规则：

- 本月收入 = 当前自然月所有收入记录金额之和。
- 本月支出 = 当前自然月所有支出记录金额之和。
- 本月结余 = 本月收入 - 本月支出。
- 预算进度 = 本月支出 / 月度预算。
- 最近账单默认显示最近 10 条。

预算状态规则：

- 支出低于预算 80%：显示正常状态。
- 支出达到预算 80% 且低于 100%：显示提醒“本月预算已使用较多”。
- 支出达到或超过预算 100%：显示预警“本月预算已超出”。

### 5.2 新增账单

用户可以新增收入或支出记录。

字段：

- 收支类型：收入 / 支出。
- 金额。
- 分类。
- 账户。
- 日期。
- 备注。

支出分类：

- 餐饮。
- 交通。
- 购物。
- 娱乐。
- 学习。
- 其他。

收入分类：

- 生活费。
- 工资。
- 兼职。
- 红包。
- 其他。

账户：

- 现金。
- 银行卡。
- 信用卡。

保存规则：

- 金额不能为空。
- 金额必须大于 0。
- 必须选择收支类型。
- 必须选择分类。
- 必须选择账户。
- 日期默认当天。
- 备注可以为空。

保存成功后：

- 账单写入 SQLite。
- 对应账户余额同步变化。
- 首页数据刷新。
- 最近账单列表刷新。

### 5.3 账单列表

账单列表用于展示已保存的收支记录。

展示内容：

- 分类名称。
- 收支类型。
- 金额。
- 账户名称。
- 日期。
- 备注。

排序规则：

- 按账单日期倒序排列。
- 同一天内按创建时间倒序排列。

第一阶段只要求展示账单列表，不强制实现筛选、编辑和删除。筛选、编辑、删除放入第二阶段。

### 5.4 预算设置

用户可以设置月度预算金额和是否开启预算提醒。

字段：

- 月度预算金额。
- 是否开启预算提醒。

保存方式：

- 使用 SharedPreferences 保存预算金额。
- 使用 SharedPreferences 保存预算提醒开关。

预算规则：

- 预算只统计支出，不统计收入。
- 预算周期为自然月。
- 新增账单后重新计算预算使用情况。

### 5.5 多账户基础支持

第一阶段支持系统默认账户。

默认账户：

- 现金。
- 银行卡。
- 信用卡。

账户字段：

- 账户 ID。
- 账户名称。
- 账户类型。
- 账户余额。

账户余额规则：

- 新增收入：对应账户余额增加。
- 新增支出：对应账户余额减少。

第一阶段不做账户新增、修改、删除。完整账户管理放入第三阶段。

## 六、非 MVP 范围

以下内容不进入第一阶段：

- 凭证上传。
- Excel 导出。
- PDF 导出。
- 周统计。
- 月度图表。
- 复杂饼图或趋势图。
- 自定义分类。
- 自定义账户。
- 分类预算。
- 信用卡账单日和还款日管理。
- 账单筛选。
- 账单编辑。
- 账单删除。
- 云同步。
- 登录注册。

## 七、页面设计

### 7.1 页面结构

第一阶段建议使用 3 个主页面：

- 首页。
- 记账。
- 我的。

首页负责展示财务概览和最近账单。

记账页负责新增收入和支出。

我的页负责预算设置和账户余额展示。

### 7.2 首页布局

首页包括：

- 本月概览区域。
- 预算进度区域。
- 最近账单列表。

关键展示字段：

- 本月收入：￥xxx。
- 本月支出：￥xxx。
- 本月结余：￥xxx。
- 预算进度：已用 xx%。
- 预算提醒文案。

### 7.3 记账页布局

记账页包括：

- 收入 / 支出切换。
- 金额输入框。
- 分类选择。
- 账户选择。
- 日期选择。
- 备注输入框。
- 保存按钮。

### 7.4 我的页布局

我的页包括：

- 月度预算设置。
- 预算提醒开关。
- 账户余额列表。

## 八、数据设计

### 8.1 bill_record 表

用于保存账单记录。

字段：

- id：INTEGER，主键，自增。
- type：TEXT，收入或支出，值为 income / expense。
- amount：REAL，金额。
- category_id：INTEGER，分类 ID。
- account_id：INTEGER，账户 ID。
- record_date：TEXT，账单日期，格式 yyyy-MM-dd。
- remark：TEXT，备注。
- create_time：INTEGER，创建时间戳。

### 8.2 category 表

用于保存分类。

字段：

- id：INTEGER，主键，自增。
- name：TEXT，分类名称。
- type：TEXT，收入分类或支出分类。
- sort_order：INTEGER，排序值。

初始化数据：

- 支出分类：餐饮、交通、购物、娱乐、学习、其他。
- 收入分类：生活费、工资、兼职、红包、其他。

### 8.3 account 表

用于保存账户。

字段：

- id：INTEGER，主键，自增。
- name：TEXT，账户名称。
- account_type：TEXT，账户类型，值为 cash / bank_card / credit_card。
- balance：REAL，账户余额。

初始化数据：

- 现金，余额 0。
- 银行卡，余额 0。
- 信用卡，余额 0。

### 8.4 SharedPreferences

用于保存预算配置。

字段：

- monthly_budget：月度预算金额。
- budget_warn_enabled：是否开启预算提醒。

## 九、技术方案

开发语言：

- Java。

本地数据库：

- SQLiteOpenHelper。

轻量配置：

- SharedPreferences。

列表展示：

- ListView + BaseAdapter。

页面结构：

- 一个主 Activity + 多个 Fragment，或多个 Activity。
- 若参考实验结构，建议使用主 Activity + Fragment 切换页面。

异步处理：

- 第一阶段数据量较小，可以直接查询。
- 后续统计和导出任务可使用 Thread + Handler。

## 十、核心流程

### 10.1 新增支出流程

1. 用户进入记账页。
2. 选择支出。
3. 输入金额。
4. 选择分类，例如餐饮。
5. 选择账户，例如现金。
6. 选择日期或使用默认日期。
7. 输入备注或留空。
8. 点击保存。
9. 系统校验字段。
10. 写入 bill_record 表。
11. 更新现金账户余额。
12. 刷新首页数据和最近账单。

### 10.2 新增收入流程

1. 用户进入记账页。
2. 选择收入。
3. 输入金额。
4. 选择分类，例如生活费。
5. 选择账户，例如银行卡。
6. 点击保存。
7. 系统校验字段。
8. 写入 bill_record 表。
9. 更新银行卡账户余额。
10. 刷新首页数据和最近账单。

### 10.3 预算预警流程

1. 用户在我的页设置月度预算。
2. 系统使用 SharedPreferences 保存预算配置。
3. 首页读取预算金额和预警开关。
4. 系统查询本月所有支出总额。
5. 计算预算使用比例。
6. 根据比例展示正常、提醒或超额状态。

## 十一、开发顺序建议

1. 创建数据库表。
2. 初始化默认分类和默认账户。
3. 完成首页静态布局。
4. 完成记账页静态布局。
5. 实现分类和账户读取。
6. 实现账单保存。
7. 实现账单列表展示。
8. 实现首页收入、支出、结余统计。
9. 实现账户余额更新。
10. 实现预算设置。
11. 实现预算进度和预警。
12. 完成输入校验和基础测试。

## 十二、验收标准

- App 启动后能进入首页。
- 首页能展示本月收入、本月支出、本月结余。
- 用户能新增一笔支出记录。
- 用户能新增一笔收入记录。
- 账单能保存到 SQLite。
- 关闭 App 后重新打开，账单仍存在。
- 最近账单列表能正确展示记录。
- 默认分类能正常选择。
- 默认账户能正常选择。
- 新增账单后账户余额能正确变化。
- 用户能设置月度预算。
- 首页能显示预算进度。
- 支出达到 80% 时能显示预算提醒。
- 支出达到或超过 100% 时能显示超额预警。
- 金额为空时不能保存。
- 金额为 0 或负数时不能保存。
- 收入和支出分类不会混用。

## 十三、交付重点

第一阶段交付重点是：

本地记账核心闭环完整，预算预警可演示，数据关闭后不丢失。
