# 阶段性开发任务

## 一、项目目标

本项目设计一款个人记账 App，面向学生、上班族和日常家庭记账用户。产品核心目标是帮助用户快速记录收入和支出，查看周期统计，控制月度预算，并支持后续扩展凭证上传、账单导出和多账户管理。

项目采用阶段化开发方式，优先完成本地记账核心闭环，再逐步扩展统计、导出、凭证和账户管理功能。

## 二、技术参考

结合上一级目录中的实验内容，项目优先参考以下技术：

- SQLite：用于账单、分类、账户、预算等本地数据存储。
- SharedPreferences：用于保存预算金额、预警开关、默认配置等轻量数据。
- ListView + 自定义 Adapter：用于账单列表、账户列表、分类列表展示。
- Glide：用于后续加载凭证图片。
- Handler + Thread：用于统计、导出等耗时任务后的界面更新。
- Gson：用于后续数据备份、导入导出结构化数据。
- OkHttp：预留给后续云同步或网络备份。
- PdfDocument：用于后续生成 PDF 账单。
- Canvas：用于后续绘制简单统计图表。

## 三、第一阶段：MVP 核心记账闭环

### 阶段目标

完成一个可本地运行、可保存数据、可查看基础统计、可进行预算预警的记账 App。

第一阶段重点保证以下核心流程完整：

新增账单 -> 保存到 SQLite -> 列表展示 -> 首页汇总 -> 预算提醒。

### 开发范围

- 新增收入和支出记录。
- 支持基础支出分类：餐饮、交通、购物、娱乐、学习、其他。
- 支持基础收入分类：生活费、工资、兼职、红包、其他。
- 支持基础账户：现金、银行卡、信用卡。
- 使用 SQLite 保存账单、分类和账户数据。
- 首页展示本月收入、本月支出、本月结余。
- 首页展示最近账单列表。
- 支持月度预算设置。
- 首页展示预算使用进度和预警状态。
- 新增账单后同步更新账户余额。

### 暂不包含

- 凭证上传。
- Excel/PDF 导出。
- 复杂图表。
- 周统计。
- 分类预算。
- 自定义分类。
- 自定义账户。
- 信用卡账期管理。
- 云同步。
- 登录注册。

### 开发任务

1. 创建 SQLite 数据库和核心数据表。
2. 初始化默认分类和默认账户。
3. 创建首页布局，展示本月收入、支出、结余。
4. 创建记账页面，支持输入金额、选择类型、分类、账户、日期和备注。
5. 实现账单保存逻辑。
6. 实现账单列表展示。
7. 实现本月收支统计。
8. 实现账户余额联动。
9. 实现预算设置页面或入口。
10. 使用 SharedPreferences 保存预算配置。
11. 实现预算进度展示和 80%/100% 预警。
12. 完成字段校验和基础测试。

### 验收标准

- App 启动后可以进入首页。
- 用户可以新增一笔收入记录。
- 用户可以新增一笔支出记录。
- 账单可以保存到 SQLite。
- 关闭 App 后重新打开，账单仍然存在。
- 首页可以展示本月收入、本月支出和本月结余。
- 最近账单列表可以展示已保存记录。
- 默认分类可以正常选择。
- 默认账户可以正常选择。
- 新增收入后账户余额增加。
- 新增支出后账户余额减少。
- 用户可以设置月度预算。
- 首页可以展示预算进度。
- 支出达到预算 80% 时显示提醒。
- 支出达到或超过预算 100% 时显示超额预警。
- 金额为空、金额为 0 或负数时不能保存。
- 收入分类和支出分类不会混用。

## 四、第二阶段：统计与体验增强

### 阶段目标

在第一阶段可用记账功能基础上，增强数据分析能力和账单管理体验。

### 开发范围

- 周统计。
- 月统计。
- 分类支出排行。
- 简单柱状图。
- 分类占比图。
- 账单筛选：按日期、分类、账户、类型筛选。
- 账单详情查看。
- 账单编辑。
- 账单删除。
- 首页最近账单展示优化。

### 开发任务

1. 实现按周统计收入和支出。
2. 实现按月统计收入和支出。
3. 实现分类支出汇总。
4. 使用 Canvas 绘制简单柱状图。
5. 实现分类支出排行列表。
6. 增加账单详情页面。
7. 增加账单编辑功能。
8. 增加账单删除功能。
9. 修改和删除账单时同步修正账户余额。
10. 增加账单筛选条件。

### 验收标准

- 周统计金额准确。
- 月统计金额准确。
- 分类支出排行准确。
- 图表可以正常展示统计结果。
- 用户可以查看账单详情。
- 用户可以编辑账单。
- 用户可以删除账单。
- 编辑或删除账单后，首页统计和账户余额同步变化。

## 五、第三阶段：扩展功能

### 阶段目标

覆盖课程设计需求中的扩展能力，包括凭证上传、账单导出和完整账户管理。

### 开发范围

- 凭证图片上传。
- 凭证图片查看。
- CSV/Excel 兼容导出。
- PDF 账单导出。
- 账户管理：新增、修改、删除账户。
- 分类管理：新增、修改、删除分类。

### 开发任务

1. 接入系统相册选择图片。
2. 将凭证图片路径保存到账单记录。
3. 使用 Glide 在详情页显示凭证图片。
4. 实现按本周、本月、全部、自定义时间范围导出账单。
5. 实现 CSV 文件导出，支持 Excel 打开。
6. 使用 PdfDocument 生成 PDF 账单。
7. 增加账户新增、修改、删除功能。
8. 增加分类新增、修改、删除功能。

### 验收标准

- 用户可以为账单上传凭证图片。
- 账单详情页可以查看凭证图片。
- 没有凭证的账单仍可正常保存。
- 用户可以导出本月账单。
- CSV 文件可以用 Excel 打开。
- PDF 文件可以正常生成和打开。
- 用户可以新增、修改和删除账户。
- 用户可以新增、修改和删除分类。

## 六、第四阶段：高级优化

### 阶段目标

提升项目完整度和可扩展性，但不作为课程设计首要交付内容。

### 开发范围

- 数据备份与恢复。
- Gson 导出 JSON 备份。
- OkHttp 预留云同步接口。
- 信用卡还款日提醒。
- 分类预算。
- 主题设置或深色模式。

### 开发任务

1. 实现 JSON 格式数据备份。
2. 实现 JSON 数据恢复。
3. 预留 OkHttp 网络同步接口。
4. 增加信用卡账单日和还款日字段。
5. 实现信用卡还款提醒。
6. 实现分类预算。
7. 增加主题配置。

## 七、快速开发原则

项目采用先核心、后扩展的快速开发策略。

第一阶段只实现最小可用产品，确保记账、保存、展示、汇总和预算提醒完整闭环。凭证上传、图表、导出和复杂账户管理不进入 MVP，以降低开发风险。

第二阶段开始增强统计和账单管理体验。

第三阶段再实现题目扩展要求中的凭证、导出和账户管理。

第四阶段作为加分项或后续优化。
